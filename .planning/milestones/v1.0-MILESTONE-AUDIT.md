## Cross-Phase Integration Check

### Framing (Phase 1) → Session (Phase 2) → Channel (Phase 3)

`encode()` and `decode()` are called in `src/channel/channel.ts` for all inbound
and outbound frames. `Session` is constructed inside `Channel.#createSession()` and
receives every inbound `DataFrame`, `CreditFrame`, `CloseFrame`, `CancelFrame`, and
`ResetFrame` via `session.receiveFrame(frame)`. The sequence-arithmetic helpers
(`seqLT`, `seqNext`) from Phase 1 are imported by `src/session/reorder-buffer.ts`
and `src/session/chunker.ts`.

**Status: WIRED**

### Session.onError → Channel.on('error') routing (Phase 4)

`src/channel/channel.ts` lines ~727–731: `#createSession()` wires
`session.onError(reason => ... #emitter.emit('error', new StreamError(code)))`.
The `mapSessionErrorCode` helper at line ~76 translates every session-level reason
(`consumer-stall`, `reorder-overflow`) to the OBS-02 typed `ErrorCode` union.
All 7 named error codes are verified in Phase 4 tests.

**Status: WIRED**

### CAPABILITY frame negotiation chain (Phase 1 types + Phase 3 channel + Phase 6 SAB + Phase 8 multiplex)

`CapabilityFrame` (Phase 1 type) is assembled in `Channel.#sendCapability()` with
`sab: this.#localCap.sab` (Phase 6), `transferableStreams: ...` (Phase 3), and
`multiplex: this.#localCap.multiplex` (Phase 8). Merging happens in
`#handleCapability()` as logical AND of both sides for each capability bit.
Feature detection runs once (`isSabCapable(endpoint)` at construction; Phase 6
`FAST-05` invariant upheld).

**Status: WIRED — all three phases extend CAPABILITY consistently**

### SAB path activation does not bypass the session layer (Phase 6)

SAB DATA frames are dispatched via `#dispatchSabFrame()`, which calls
`this.#session.receiveFrame(frame)` — the same path as postMessage DATA frames.
The SAB ring carries raw payload + seq + chunkType; reassembly, reorder buffering,
and credit management run identically on both paths. The ring is only used for
DATA frame payloads; CREDIT frames and all control frames continue over postMessage.

**Status: WIRED — SAB ring is a transport-layer fast path, not a session bypass**

### RelayBridge uses Channel raw-frame hooks (Phase 7 + Phase 3)

`src/relay/bridge.ts` calls `upstream.onRawDataFrame()`, `upstream.onRawControlFrame()`,
`downstream.onRawDataFrame()`, `downstream.onRawControlFrame()`, and
`channel.sendRawFrame()`. All five methods are defined in `src/channel/channel.ts`
(lines 635, 648, 660). Raw-frame hooks fire BEFORE session delivery in the inbound
handler (channel.ts line ~377), ensuring relay forwards frames and the local session
continues independently without double-counting.

**Status: WIRED**

### Multiplex Map<streamId, Session> does not break Phase 4 stats or Phase 7 relay (Phase 8 + Phase 4 + Phase 7)

`channel.stats()` iterates `this.#sessions.values()` to build the `streams[]`
array (line 699). The relay bridge never calls `openStream()` or `onStream()` on the
relay channels — it operates exclusively through raw-frame hooks, so the session Map
is empty on relay channels, and `stats().streams` correctly returns `[]`.
Phase 4 `bytesSent` / `bytesReceived` counters are incremented in `sendFrame` and
`onmessage`, which are called for both multiplex and relay paths.

**Status: WIRED — no conflicts between Phase 4 OBS-01, Phase 7 relay, and Phase 8 multiplex**

### E2E fixture pages consume the built library (Phase 9 + Phase 10)

E2E fixture pages (`e2e/fixtures/pages/*.html`) import from `"/dist/index.js"` —
the real built library artifact. The fixture HTTP server serves the repo root,
so `dist/index.js` is the tsdown-built output from `src/index.ts`.
Examples (`examples/0N/main.ts`) import from `"iframebuffer"`, which resolves to
`file:../..` (the root `package.json` with `exports["."]: "./dist/index.js"`).

**Status: WIRED — both E2E tests and examples consume the production build**

### CI workflow covers all required gates

`ci.yml` steps verified in Phase 1 VERIFICATION.md and updated in Phase 9/10:
lint → typecheck → unit tests (340/340) → E2E local (chromium+firefox) →
E2E CI (chromium+firefox+webkit via `--with-deps`) → version-sync check.
`publish.yml` has a dry-run job (Phase 10) with OIDC `id-token: write`.

**Status: WIRED**
