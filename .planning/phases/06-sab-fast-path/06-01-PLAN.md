Create `src/transport/sab-ring.ts` implementing an SPSC ring buffer over `SharedArrayBuffer`.

Public API:
```ts
export interface SabRingView { sab: SharedArrayBuffer; capacity: number; }

export function allocSabRing(capacity: number): SabRingView;

export class SabRingProducer {
  constructor(view: SabRingView);
  async write(payload: Uint8Array, seq: number, chunkType: number, timeoutMs?: number): Promise<boolean>; // false on timeout
  writeTerminator(): void; // length=0 + notify
  close(): void; // sets flags bit 0
}

export class SabRingConsumer {
  constructor(view: SabRingView);
  async read(timeoutMs?: number): Promise<{ payload: Uint8Array; seq: number; chunkType: number } | null>; // null on closed/terminator
  close(): void;
}
```

Header layout (per RESEARCH.md):
- Int32Array view of first 64 bytes: [0]=head, [1]=tail, [2]=flags, [3]=capacity, [4..15]=reserved
- Uint8Array view of bytes 64..64+capacity = payload

Frame layout in ring:
- [u32 length][u32 seq][u32 chunkType][payload]
- `length=0` = terminator (closed)
- `length=0xFFFFFFFF` = padding-to-wrap marker (producer writes when a frame wouldn't fit before the ring end; consumer sees it and restarts at offset 0)

Producer `write`:
1. Compute required bytes (12 header + payload.length)
2. Loop: check available space = capacity - (head - tail). If insufficient, `Atomics.waitAsync(int32View, 1, tail)` until tail changes OR timeout
3. If wrap needed (current write pos + required bytes > capacity), write padding marker, advance head, re-check space, write at offset 0
4. Memcpy header + payload using `Uint8Array.set` / `Int32Array.store`
5. `Atomics.store(int32View, 0, newHead)`, `Atomics.notify(int32View, 0, 1)`

Consumer `read`:
1. Check available = head - tail. If 0, `Atomics.waitAsync(int32View, 0, head)` until head changes OR timeout
2. Read length at current position
3. If length === 0: closed, return null
4. If length === 0xFFFFFFFF: skip to offset 0, re-read
5. Otherwise: read seq, chunkType, copy payload
6. `Atomics.store(int32View, 1, newTail)`, `Atomics.notify(int32View, 1, 1)`
7. Return `{ payload, seq, chunkType }`

Create `src/transport/sab-capability.ts`:
```ts
export function isSabCapable(endpoint?: PostMessageEndpoint): boolean {
  if (typeof SharedArrayBuffer === 'undefined') return false;
  if (typeof Atomics === 'undefined' || typeof Atomics.waitAsync !== 'function') return false;
  const coi = (globalThis as { crossOriginIsolated?: boolean }).crossOriginIsolated;
  if (coi === false) return false; // strict false only; undefined in Node means "ok"
  const caps = (endpoint as unknown as { capabilities?: { sabCapable?: boolean } })?.capabilities;
  if (caps?.sabCapable === false) return false;
  return true;
}
```

<action>
Extend `ChannelOptions` with:
```ts
sab?: boolean;        // caller opt-in (default false)
sabBufferSize?: number; // default 1_048_576 (1 MB)
```

In Channel constructor:
1. Compute local `sab` capability via `isSabCapable(endpoint)` AND `options.sab === true`
2. Include `sab: localSab` in the outbound CAPABILITY frame
3. On inbound CAPABILITY, compute merged `sab = localSab && remoteSab`
4. If merged `sab: true`: initiator (alphabetically-smaller channelId or caller-flagged `initiator: true` option) allocates SAB ring via `allocSabRing(sabBufferSize)` and sends a dedicated postMessage:
   ```
   endpoint.postMessage({ __ibf_sab_init__: true, sab: sharedArrayBuffer, bufferSize }, [sharedArrayBuffer])
   ```
   Note: SAB doesn't transfer; it SHARES. Both sides hold a reference to the same memory. Transfer list is empty (passing SAB doesn't detach).
5. The peer receives the `__ibf_sab_init__` control via the existing onmessage handler, detects the special shape, and constructs its own `SabRingConsumer` on the same SAB.
6. After SAB_INIT is acknowledged (peer sends back `{ __ibf_sab_init_ack__: true }`), Channel flips the `#sabReady` flag.
7. `sendFrame(frame)`: if frame.type === 'DATA' && `#sabReady`: write via `SabRingProducer.write(encodedPayload, seq, chunkTypeNum)`; else postMessage path as before.
8. Consumer loop started on SAB_INIT_ACK: `async () => { while (!closed) { const msg = await consumer.read(timeoutMs=30_000); if (msg === null) break; this.#dispatchDataFrame(msg); } }`
9. On `channel.close()`: producer.writeTerminator(), dispose consumer, clear #sabReady. Add to #disposers array.

Error paths:
- SAB_INIT fails (peer rejects or postMessage fails) → log via `#emitter.emit('error', ...)` with a new ErrorCode `SAB_INIT_FAILED` and continue on postMessage
- Consumer read timeout (30 s) → emit `CHANNEL_DEAD` (same as heartbeat)
- Any SAB_INIT or ring error → always fall back to postMessage, never lose data

Add `SAB_INIT_FAILED` to `ErrorCode` union in `src/types.ts`.
