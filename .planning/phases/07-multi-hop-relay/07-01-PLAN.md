## Objective

Implement a relay bridge that forwards streams between two Channels without reassembly, with credit forwarding, bounded memory, and cancel/reset propagation < 100 ms.

## Tasks

### Task 1: Channel raw-frame hooks

<read_first>
- .planning/phases/07-multi-hop-relay/07-CONTEXT.md
- .planning/phases/07-multi-hop-relay/07-RESEARCH.md
- src/channel/channel.ts (current structure, frame routing, emitter)
- src/framing/types.ts (Frame types)
</read_first>

<action>
Add three public hooks to `Channel`:

```ts
onRawDataFrame(cb: (frame: DataFrame) => void): () => void;
onRawControlFrame(cb: (frame: Frame) => void): () => void;
sendRawFrame(frame: Frame, transfer?: Transferable[]): void;
```

Implementation:
- `#rawDataHandlers: Set<(frame: DataFrame) => void>`, `#rawControlHandlers: Set<(frame: Frame) => void>`
- Inside the inbound `onmessage` handler, after `decode(msg)` succeeds, fan the decoded frame out to the relevant raw-handler set in parallel with session delivery. Do NOT replace session delivery.
- `sendRawFrame` calls `this.#endpoint.postMessage(encode(frame), transfer ?? [])` directly, bypassing the session layer. Increments stats counters `framesByType` as normal.
- Returned closure removes the handler from the set.

Add a short unit test in `tests/unit/channel/channel.test.ts` (or a new file) that creates a Channel, registers `onRawDataFrame`, has the peer send a DATA frame via a test harness, asserts the callback fires and receives the frame shape.
</action>
