Phase 6 is SAB Fast Path (not WASM per se). The WASM question is: does the JS transferable path hit a measurable ceiling that only WASM can break through? If throughput is still scaling with payload size and p99 latency is not wall-flat at large payloads, the JS path has headroom.

Bottleneck heuristics:
- CPU-bound: p99/p50 ratio <= 1.5 AND throughput plateaus at large sizes → framing overhead is the bottleneck (WASM framing would help)
- GC-bound: p99/p50 ratio > 3× at 16MB → GC pauses are the bottleneck (WASM ring buffer would help)
- Channel-bound: throughput scales linearly with payload size → postMessage serialization is the bottleneck (WASM cannot help, SAB would)
- Latency-bound: p99 at 1MB ~ p99 at 64KB → round-trip overhead dominates small payloads (WASM cannot help)
