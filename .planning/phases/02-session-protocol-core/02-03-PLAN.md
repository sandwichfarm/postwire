  SESS-04 behaviors (describe 'Chunker split'):
  - BINARY_TRANSFER: ArrayBuffer of exactly maxChunkSize → 1 chunk, isFinal=true, transfer=[ab]
  - BINARY_TRANSFER: ArrayBuffer of maxChunkSize+1 → 2 chunks; first isFinal=false, second isFinal=true
  - BINARY_TRANSFER: ArrayBuffer of 3×maxChunkSize → 3 chunks all produced; seqNums are consecutive
  - BINARY_TRANSFER: seqNums advance correctly using seqNext across chunks (not raw +1)
  - BINARY_TRANSFER: each ChunkResult.transfer contains the ArrayBuffer slice (not the full original)
  - STRUCTURED_CLONE: any non-ArrayBuffer value → 1 chunk, chunkType='STRUCTURED_CLONE', transfer=[]
  - STRUCTURED_CLONE: isFinal=true (single chunk), seqNum advances

  SESS-04 invariant (describe 'metadata-before-transfer invariant'):
  - After split() is called on an ArrayBuffer, the ORIGINAL ArrayBuffer reference must NOT be
    in any frame payload of the first N-1 chunks (each chunk gets its own slice)
  - byteLength of each chunk frame is capturable BEFORE any transfer would detach it
  - This is verified by checking that ChunkResult frames have the expected byteLength and that
    re-reading metadata fields from the frame object returns correct values (not 0)

  SESS-04 behaviors (describe 'Chunker reassemble'):
  - reassemble() returns null for all non-isFinal chunks (accumulates in map)
  - reassemble() returns the concatenated ArrayBuffer when isFinal=true chunk arrives
  - reassemble() clears the map entry after returning the complete payload
  - reassemble() handles multi-chunk reassembly: 3 chunks → returns combined buffer on chunk 3
  - reassemble() for STRUCTURED_CLONE: returns payload directly on the isFinal=true frame
  - out-of-order reassembly test: the chunker assumes frames arrive in order for reassembly
    (reorder buffer handles ordering; chunker just accumulates by streamId)

  Sequence number tests (describe 'sequence numbers'):
  - First chunk from a new Chunker(initSeq=0) gets seqNum=0 (the initSeq itself, not seqNext)
  - Second chunk gets seqNum=seqNext(0)=1
  - Wraparound: initSeq=0xFFFFFFFE, 3 chunks → seqNums are [0xFFFFFFFE, 0xFFFFFFFF, 0x00000000]

  ```typescript
  // src/session/chunker.ts
  import { seqNext } from '../transport/seq.js';
  import { FRAME_MARKER } from '../framing/types.js';
  import type { DataFrame, ChunkType } from '../framing/types.js';

  export interface ChunkerOptions {
    channelId: string;
    streamId: number;
    maxChunkSize?: number;   // default: 65536 (64 KB)
  }

  export interface ChunkResult {
    frame: DataFrame;
    transfer: ArrayBuffer[];  // non-empty for BINARY_TRANSFER only
  }

  export class Chunker {
    #lastSeq: number;
    readonly #channelId: string;
    readonly #streamId: number;
    readonly #maxChunkSize: number;
    // Reassembly map: streamId → accumulated ArrayBuffer chunks
    // (keyed by streamId since Chunker is per-stream, but map allows future multi-chunk tracking)
    readonly #reassemblyBufs = new Map<number, ArrayBuffer[]>();

    constructor(initSeq: number, opts: ChunkerOptions) {
      this.#lastSeq = (initSeq >>> 0) - 1 >>> 0;  // so first seqNext() yields initSeq
      // Actually: store initSeq - 1 such that the first seqNext call yields initSeq.
      // Better: use a flag for first call, OR store initSeq directly and use it on first call.
      // SIMPLEST: store #nextSeq = initSeq >>> 0 and use it directly then advance.
      this.#lastSeq = initSeq >>> 0;  // will be used as-is on first chunk, then seqNext'd
      this.#channelId = opts.channelId;
      this.#streamId = opts.streamId;
      this.#maxChunkSize = opts.maxChunkSize ?? 65536;
    }
  ```

  IMPORTANT NOTE on sequence number design: the first chunk should have seqNum=initSeq. Use a counter `#nextSeq` initialized to `initSeq >>> 0`. Each chunk consumes `#nextSeq` then advances with `seqNext`:

  Full split() implementation:

  ```typescript
  split(payload: unknown, chunkType: 'BINARY_TRANSFER' | 'STRUCTURED_CLONE'): ChunkResult[] {
    if (chunkType === 'BINARY_TRANSFER') {
      const ab = payload instanceof ArrayBuffer
        ? payload
        : (payload as ArrayBufferView).buffer;
      const total = ab.byteLength;  // STEP 1: capture BEFORE any transfer
      const results: ChunkResult[] = [];
      let offset = 0;

      do {
        // STEP 2: capture all metadata before creating the slice reference
        const chunkSize = Math.min(this.#maxChunkSize, total - offset);
        const isFinal = offset + chunkSize >= total;
        const seq = this.#nextSeq;
        this.#nextSeq = seqNext(this.#nextSeq);

        // STEP 3: slice (each slice is a new ArrayBuffer — the original is NOT transferred)
        const slice = ab.slice(offset, offset + chunkSize);

        // STEP 4: build frame with the slice as payload
        const frame: DataFrame = {
          [FRAME_MARKER]: 1,
          channelId: this.#channelId,
          streamId: this.#streamId,
          seqNum: seq,
          type: 'DATA',
          chunkType: 'BINARY_TRANSFER',
          payload: slice,
          isFinal,
        };

        // STEP 5: transfer list contains the slice, not the original
        results.push({ frame, transfer: [slice] });
        offset += chunkSize;
      } while (offset < total);

      return results;
    }

    // STRUCTURED_CLONE: single chunk, no transfer
    const seq = this.#nextSeq;
    this.#nextSeq = seqNext(this.#nextSeq);
    const frame: DataFrame = {
      [FRAME_MARKER]: 1,
      channelId: this.#channelId,
      streamId: this.#streamId,
      seqNum: seq,
      type: 'DATA',
      chunkType: 'STRUCTURED_CLONE',
      payload,
      isFinal: true,
    };
    return [{ frame, transfer: [] }];
  }
  ```

  Full reassemble() implementation:

  ```typescript
  reassemble(frame: DataFrame): unknown | null {
    if (frame.chunkType === 'STRUCTURED_CLONE') {
      return frame.isFinal ? frame.payload : null;
    }

    // BINARY_TRANSFER: accumulate slices
    const streamId = frame.streamId;
    if (!this.#reassemblyBufs.has(streamId)) {
      this.#reassemblyBufs.set(streamId, []);
    }
    this.#reassemblyBufs.get(streamId)!.push(frame.payload as ArrayBuffer);

    if (!frame.isFinal) return null;

    // Concatenate all accumulated ArrayBuffers
    const bufs = this.#reassemblyBufs.get(streamId)!;
    this.#reassemblyBufs.delete(streamId);
    const totalLen = bufs.reduce((acc, b) => acc + b.byteLength, 0);
    const result = new ArrayBuffer(totalLen);
    const view = new Uint8Array(result);
    let pos = 0;
    for (const buf of bufs) {
      view.set(new Uint8Array(buf), pos);
      pos += buf.byteLength;
    }
    return result;
  }
  ```

  Key rules:
  - `ab.slice()` creates a COPY — the original `ab` is NOT transferred; this is the safe pattern for chunker (we transfer each slice independently)
  - `ab.byteLength` is captured BEFORE any slice operations (though slicing doesn't detach the original, the comment in the code must be explicit about ordering)
  - All metadata fields (isFinal, seq, chunkSize) are local variables captured before the frame object is constructed
  - NO reading from `slice` or `ab` after a call that would transfer (in this chunker, we never call the actual postMessage — that's Transport's job. Chunker just produces frames + transfer lists)
  - seqNext() for every seq increment — never raw +1
